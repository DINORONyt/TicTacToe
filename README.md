# Курсовая работа: Разработка игрового проекта «Крестики-нолики»

### Описание проекта
Данный проект представляет собой приложение на языке C# с графическим интерфейсом Windows Forms, разработанное в рамках курсовой работы по дисциплине «Основы программирования». Приложение реализует классическую настольную игру «Крестики-нолики» с поддержкой двухпользовательского режима, строгой проверкой очередности ходов, автоматическим определением победы и ничьей, а также визуальной подсветкой выигрышной линии. Бизнес-логика полностью вынесена в отдельный класс `Game`, интерфейс и код разделены по принципу слабой связанности, проект покрыт модульными тестами на фреймворке xUnit.

### Функциональность

**Основная механика** — Поочерёдная игра двух игроков  
Игроки по очереди размещают свои символы (крестики `X` и нолики `O`) на поле 3×3. Первый ход выполняет игрок, выбравший крестики. Запрещено размещать символ в занятой клетке или делать ход вне своей очереди.

**Система определения победителя**  
Программа автоматически проверяет 8 возможных выигрышных комбинаций (3 строки, 3 столбца, 2 диагонали). При обнаружении победы:
- Выводится сообщение о победителе в статусном поле
- Выигрышная линия подсвечивается красным цветом для наглядности

**Определение ничьей**  
Если все клетки поля заполнены, но победитель не определён, игра завершается ничьей с соответствующим уведомлением пользователя.

**Защита от некорректных действий**  
- Блокировка возможности сделать два хода подряд одним игроком
- Отключение кнопок выбора игрока после начала партии
- Запрет ручного редактирования статусного поля (`ReadOnly`)
- Валидация ввода на уровне интерфейса и бизнес-логики

### Использование приложения

При запуске открывается окно с игровым полем и панелью управления:

**Начало игры**
1. Выберите начального игрока: нажмите «Ход крестиков» (X) или «Ход ноликов» (O)
2. Статусное поле отобразит: «Ходит X» или «Ходит O»

**Выполнение хода**
1. Кликните по свободной клетке поля 3×3
2. Символ текущего игрока отобразится в выбранной клетке
3. Программа автоматически проверит победу или ничью
4. При продолжении игры ход передаётся следующему игроку

**Завершение игры**
- При победе: статус покажет «Победа: X» (или O), выигрышная линия подсветится красным
- При ничьей: статус покажет «Ничья!»
- Для новой партии: нажмите «Очистить поле» — игра сбросится к начальному состоянию

### Примеры работы

**Сценарий 1: Победа по диагонали**
```
Ход 1: Игрок X → клетка (0,0)
Ход 2: Игрок O → клетка (0,1)
Ход 3: Игрок X → клетка (1,1)
Ход 4: Игрок O → клетка (1,0)
Ход 5: Игрок X → клетка (2,2) → ПОБЕДА!
```
Результат: Статус «Победа: X», клетки (0,0), (1,1), (2,2) подсвечены красным.

**Сценарий 2: Попытка двойного хода**
```
Игрок X делает ход → ход принят
Игрок X пытается сделать ещё один ход → ход отклонён, статус не меняется
```
Результат: Игра ожидает хода игрока O, нарушение очередности заблокировано.

**Сценарий 3: Ничья**
```
Все 9 клеток заполнены, победная линия не образована
```
Результат: Статус «Ничья!», игра завершена.

### Обработка ошибок

Приложение включает многоуровневую систему валидации:

**Уровень интерфейса (Form1)**
- Проверка пустоты клетки перед ходом
- Блокировка кнопок выбора игрока после начала игры
- Запрет редактирования статусного поля

**Уровень бизнес-логики (Game)**
- Проверка границ поля (0–2) в методе `MakeMove()`
- Проверка соответствия игрока очередности (`player == CurrentPlayer`)
- Проверка занятости клетки

**Сообщения пользователю**
- Информационные сообщения в `textBox1`: «Ходит X», «Победа: O», «Ничья!»
- Визуальная подсветка: красный цвет для выигрышной линии
- Все сообщения программные, исключён ручной ввод

### Тестирование
Тестирование платформы проводилось в среде **Windows 10 и новее** (включая Windows 11).
Проект включает 24 модульных теста на фреймворке **xUnit**, покрывающих:
- Инициализацию и сброс состояния игры
- Валидацию ходов (границы, занятость, очередность)
- Все 8 выигрышных комбинаций (параметризованные тесты)
- Определение ничьей и приоритет победы над ничьей
- Корректность срабатывания события `OnWin`
- Интеграционный сценарий полной партии
